'学习笔记-2016-10-23'

Posted on 2016-10-23

首先明确一个问题，JAVA之所能够跨平台运行，是因为程序都是运行在JAVA虚拟机上
所有的JAVA程序都运行在JVM中

那么可以理解为由JVM来管理JAVA程序的内存

基本数据类型+引用数据类型

在JVM中划分5块内存区
1.寄存器（Program Counter Register）
2.本地方法区（系统相关）
3.方法区（数据区，共享区，ShareData）
4.栈
method -> 栈
存储的都是局部变量（Var）,也在栈区
变量所属的作用域一旦结束，该变量就自动释放
5.堆
存储的是数组和对象（数组就是对象）
堆中存储实体
new 建立的对象都在堆中。
布尔值默认为假
特点：
1.每一个实体都有一个首地址值
2.堆内存中的每一个变量都有默认初始化值
根据类型不同而不同。

定义类：定义成员变量filed（属性）和成员函数method（行为）

field：
A data member of a class. Unless specified otherwise, a field is not static.

class variable：
A data item associated with a particular class as a whole–not with particular instances of the class. Class variables are defined in class definitions.
Also called a static field. See also instance variable.

instance variable：
Any item of data that is associated with a particular object. Each instance of a class has its own copy of the instance variables defined in the class.
Also called a field. See also class variable.

local variable：
A data item known within a block, but inaccessible to code outside the block.
For example, any variable defined within a method is a local variable and can’t be used outside the method

Car c1 = new Car();
Car c2 = c1;

c1.num=1;
c2.color="red";

System.out.println(c1);
System.out.println(c2);

成员变量和局部变量的区别：
1.
成员变量定义在类中，整个类都可以访问
局部变量定义在函数，语句，局部代码块中，只在所属区域有效
2.
成员变量存在与堆内存的对象中
成员变量存在与栈内存的方法中
3.
成员变量随着对象的创建而存在，随着对象的消失而消失
局部变量随着所属区域的执行而存在，随着所属区域的结束而释放
4.
成员变量都有默认初始化值
局部变量没有默认初始化值
5.
当局部变量和成员变量重名时，在成员函数的内部会优先调用局部变量
若想调用成员变量，用 this.Var

6.成员变量也成为实例变量，即对象存在，该变量才存在。

匿名对象：
1.当对象对方法仅一次调用的时候，简化成匿名对象
new Car().run;
2.匿名对象可以作为实际参数进行传递

show(new Car())

构造函数：对象创建后，就会调用与之对应的构造函数，对对象进行初始化。
什么时候定义构造函数：在描述事物时该事物一存在就具备一些内容
这些内容都定义在构造函数中。

构造函数特点：
1.函数名与类名相同
2.不用定义返回值类型
3.没有具体的返回值

多个构造函数在类中以重载的形式来体现
构造函数首字母大写，成员函数首字母小写

this
所在函数所属对象的引用

简单说：哪个对象调用了this所在的函数，this就代表哪个对象

为什么main函数要带String数组参数 可以接受在程序编译前的参数传递

例：java mainDemo（类名） haha hehe xixi（参数）

static:
作用：修饰符，用于修饰成员（成员变量和成员函数）
被修饰后的成员具备以下特点：
1.随着类的加载而加载
2.优先于对象存在，因为static的成员随着类的加载就已经存在了。
3.static 修饰的成员可以被所有对象所共享
4.static修饰的数据是共享数据，对象中的存储的是特有数据。
Class Person{
static Country = null；
}
Person.Country
使用注意：
1.静态方法只能访问静态成员
2.静态方法中不可以写this super关键字
3.主函数是静态的

成员变量和静态变量的区别？
1.变量的生命周期不同
成员变量随着对象的创建而存在，随着对象的被回收而释放
静态变量随着类的加载而存在，随着类的消失而消失。
（一般情况下，虚拟机结束，类就消失，跟GC有关）
2.调用方式不同。
成员变量只能被对象调用。
静态变量可以被对象调用，还可以被类名调用。
3.别名不同。
成员变量也称为实例变量。
静态变量称为类变量。
4.数据存储位置不同
成员变量数据存储在堆内存中，所有也叫对象的特有数据。
静态变量数据存储在方法区的（方法区中的Runtime Constant Pool）

注意事项：
1.静态方法只能访问静态成员（非静态既可以访问静态也可以访问非静态）
2.静态方法中不能使用This或者Super
3.主函数是静态的
解释：如果main()方法没有使用static修饰符，那么编译不会出错，但是如果你试图执行该程序将会报错，提示main()方法不存在。因为包含main()的类并没有实例化（即没有这个类的对象），所以其main()方法也不会存。而使用static修饰符则表示该方法是静态的，不需要实例化即可使用。
main方法是一个程序的入口，如果写成非静态的，那么就必须实例化一个对象再来调用它，既然是入口这样肯定是不可以的；静态方法是属于类的，直接用类名就可以调用
static方法不需要创建对象 可以直接访问 这样main方法就不需要做特殊处理
static方法在静态数据区中，这样就可以直接调用不需要每次调用都要重新生成。
静态函数在调用时不用实例化
静态,一上来就装入内存

简单来说一句话总结：Main函数是入口，需要直接被加载，无法创建对象，所以static修饰。

主函数特殊之处：
1.格式固定
2.被JVM所识别和调用

public static void main(String[] args)//new String[0] JVM往main函数中传的参数
分析：
public：因为权限最大
static：不需要对象，直接用主函数所属类名调用
void：没有返回值
main：函数名，不是关键字，只是一个JVM识别的固定的名字
string[] args: 主函数参数列表，是一个数组类型的参数，而且元素都是字符串类型
接受在编译时的参数
