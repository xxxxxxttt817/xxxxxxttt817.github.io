静态变量什么时候用？
1.静态变量
当分析对象中所具备的成员变量的值都是相同的。
这是这个成员就可以被静态修饰。
只要数据在对象中都是不同的，就是对象的特有数据，必须存储在对象中，非静态
如果是相同的数据，对象不需要做修改，只需要使用即可，不需要存储在对象中，定义成静态。
2.静态函数（功能）
函数是否用静态修饰，就参考一点，就是该函数功能是否有访问到对象中的特有数据
简单点说，从源代码看，该功能是否需要访问非静态的成员变量，如果有需要，该功能就是非静态。
如果不需要，就可以将该功能定义成静态
当然，也可以定义成非静态，但是非静态需要被对象调用，而仅创建对象调用非静态的没有访问特有数据额方法，该对象的创建是没有意思的

静态函数只能调用静态成员变量，非静态函数则可以调用普通成员和静态成员。

构造代码块，可以给所有对象进行初始化
{} 创建对象时调用
构造函数，给对象进行针对性的初始化

静态代码块
static{}
随着类的加载而执行。而且只执行一次
作用：用于给类初始化

局部代码块（限定局部变量生成周期），构造代码块 ，静态代码块

执行顺序：静态代码块->构造代码块->构造函数

测试：
public class Person {
private String name;
private static String country;

{               //构造代码块，在创建对象前执行
    show();
}

Person(){       //创建对象时调用构造函数
    country = "cn";
}

static{         //静态代码块在类加载时执行
    country = "USA";
}

public void show(){
    System.out.println(country);
}


public static void main(String[] args){
    Person p1 = new Person();
    p1.show();
}

}

//打印结果：
USA

cn

继承
好处：
1.提高代码复用性
2.让类与类之间产生了关系，给第三特征多态提供了前提

单继承：即只有一个直接飞父类
多继承：通过“多实现”的方式体现

当要使用一个继承体系时，
1.查看该体系的顶层类，了解该体系的基本功能。
2.创建体系中的最子类对象，完成功能的使用。

/
在子父类中，成员的特点体现（所有的成员）
1.成员变量
2.成员函数
3.构造函数
*/

/1.成员变量
当本类的成员和局部变量同名用this区分
当子父类中的成员变量同名用super区分
this和super用户很相似
this代表一个本类对象的引用，super代表一个父类的空间
子类不可访问父类的私有成员变量，要想访问，通过get、set方法
*/

/*2.成员函数
当子父类中出现成员函数一模一样的情况，会运行子类的函数
这种现象，称为覆盖操作，这是函数在子父类中的特性。
函数两个特性，一个叫重载，一个叫覆盖。
1.重载，在同一个类中 overload，多个同名函数，通 过参数列表实现
2.覆盖，在子类中。覆盖也称为重写，override

覆盖主要实现：
1.子类方法覆盖父类方法时，子类权限必须要大于等于父类的权限
2.静态只能覆盖静态，或被静态覆盖
**/

什么时候使用覆盖操作：
当对一个类进行子类的扩展时，子类需要保留父类的功能声明，
但是需要定义子类中该功能的特有内容时，就使用覆盖操作完成。

子父类中构造函数的特点
在子类构造对象时，发现，访问子类构造函数时，
父类也运行了。
为什么呢？
原因是：在子类的构造函数第一行有一个默认的隐式语句。super（）；
注意：
1.构造函数不能覆盖
2.构造函数

为什么子类实例化的时候要访问父类中的构造方法？
那是因为子类继承了父类，获取到了父类中的内容（属性），所以在使用父类内容之前
要先看父类是如何对自己内容进行初始化的

所以子类在构造对象时，必须访问父类中的构造函数。
为什么完成这个必须的动作，就在子类得到构造函数中加入了super（）语句

如果父类中没有定义空参数构造函数，那么子类的构造函数必须用super明确要调用父类中的哪个构造函数。
同时子类构造函数中如果使用this调用了本类构造函数是，
那么super就没有了，因为super和this都只能定义第一行，所有只能有一个。
但是可以保证的是，子类中肯定会有其他的构造函数访问父类的构造函数。

注意：
1.super 语句必须要定义在子类构造函数的第一行。因为父类的初始化动作要先完成。
2.当子父类出现重名函数时，并且该函数在父类的构造函数中调用，则在子函数的构造函数中会去优先调用子函数的该函数
例：

class Fu {
Fu(){
show();
}
void show(){}
}

class Zi extends Fu {
int num = 5;
Zi(){
show();
}

void show(){
    System.out.println(num);
}

}

public class ExtendsDemo2 {
public static void main(String[] args) {

    Zi z = new Zi();

}

}
//输出结果：
0
5

一个对象实例化过程：
Person p = new Person()
1.JVM会读取指定的路径下的Person.class文件，并加载进内存，
并会先加载Person的父类（如果有直接的父类的情况下），

4.调用对应的构造函数进行初始化。
5.在构造函数中，第一行会先调用父类中构造函数进行初始化。
6.父类初始化完毕后，在对子类的属性进行显示初始化。
7.在进行子类构造函数的特定初始化
8.初始化完毕后，将地址赋值给引用变量。

首先明确一个问题，JAVA之所能够跨平台运行，是因为程序都是运行在JAVA虚拟机上
所有的JAVA程序都运行在JVM中

那么可以理解为由JVM来管理JAVA程序的内存

基本数据类型+引用数据类型

在JVM中划分5块内存区
1.寄存器（Program Counter Register）
2.本地方法区（系统相关）
3.方法区（数据区，共享区，ShareData）
4.栈
method -> 栈
存储的都是局部变量（Var）,也在栈区
变量所属的作用域一旦结束，该变量就自动释放
5.堆
存储的是数组和对象（数组就是对象）
堆中存储实体
new 建立的对象都在堆中。
布尔值默认为假
特点：
1.每一个实体都有一个首地址值
2.堆内存中的每一个变量都有默认初始化值
根据类型不同而不同。

定义类：定义成员变量filed（属性）和成员函数method（行为）

field：
A data member of a class. Unless specified otherwise, a field is not static.

class variable：
A data item associated with a particular class as a whole–not with particular instances of the class. Class variables are defined in class definitions.
Also called a static field. See also instance variable.

instance variable：
Any item of data that is associated with a particular object. Each instance of a class has its own copy of the instance variables defined in the class.
Also called a field. See also class variable.

local variable：
A data item known within a block, but inaccessible to code outside the block.
For example, any variable defined within a method is a local variable and can’t be used outside the method

Car c1 = new Car();
Car c2 = c1;

c1.num=1;
c2.color="red";

System.out.println(c1);
System.out.println(c2);

成员变量和局部变量的区别：
1.
成员变量定义在类中，整个类都可以访问
局部变量定义在函数，语句，局部代码块中，只在所属区域有效
2.
成员变量存在与堆内存的对象中
成员变量存在与栈内存的方法中
3.
成员变量随着对象的创建而存在，随着对象的消失而消失
局部变量随着所属区域的执行而存在，随着所属区域的结束而释放
4.
成员变量都有默认初始化值
局部变量没有默认初始化值
5.
当局部变量和成员变量重名时，在成员函数的内部会优先调用局部变量
若想调用成员变量，用 this.Var

6.成员变量也成为实例变量，即对象存在，该变量才存在。

匿名对象：
1.当对象对方法仅一次调用的时候，简化成匿名对象
new Car().run;
2.匿名对象可以作为实际参数进行传递

show(new Car())

构造函数：对象创建后，就会调用与之对应的构造函数，对对象进行初始化。
什么时候定义构造函数：在描述事物时该事物一存在就具备一些内容
这些内容都定义在构造函数中。

构造函数特点：
1.函数名与类名相同
2.不用定义返回值类型
3.没有具体的返回值

多个构造函数在类中以重载的形式来体现
构造函数首字母大写，成员函数首字母小写

this
所在函数所属对象的引用

简单说：哪个对象调用了this所在的函数，this就代表哪个对象

为什么main函数要带String数组参数 可以接受在程序编译前的参数传递

例：java mainDemo（类名） haha hehe xixi（参数）

static:
作用：修饰符，用于修饰成员（成员变量和成员函数）
被修饰后的成员具备以下特点：
1.随着类的加载而加载
2.优先于对象存在，因为static的成员随着类的加载就已经存在了。
3.static 修饰的成员可以被所有对象所共享
4.static修饰的数据是共享数据，对象中的存储的是特有数据。
Class Person{
static Country = null；
}
Person.Country
使用注意：
1.静态方法只能访问静态成员
2.静态方法中不可以写this super关键字
3.主函数是静态的

成员变量和静态变量的区别？
1.变量的生命周期不同
成员变量随着对象的创建而存在，随着对象的被回收而释放
静态变量随着类的加载而存在，随着类的消失而消失。
（一般情况下，虚拟机结束，类就消失，跟GC有关）
2.调用方式不同。
成员变量只能被对象调用。
静态变量可以被对象调用，还可以被类名调用。
3.别名不同。
成员变量也称为实例变量。
静态变量称为类变量。
4.数据存储位置不同
成员变量数据存储在堆内存中，所有也叫对象的特有数据。
静态变量数据存储在方法区的（方法区中的Runtime Constant Pool）

注意事项：
1.静态方法只能访问静态成员（非静态既可以访问静态也可以访问非静态）
2.静态方法中不能使用This或者Super
3.主函数是静态的
解释：如果main()方法没有使用static修饰符，那么编译不会出错，但是如果你试图执行该程序将会报错，提示main()方法不存在。因为包含main()的类并没有实例化（即没有这个类的对象），所以其main()方法也不会存。而使用static修饰符则表示该方法是静态的，不需要实例化即可使用。
main方法是一个程序的入口，如果写成非静态的，那么就必须实例化一个对象再来调用它，既然是入口这样肯定是不可以的；静态方法是属于类的，直接用类名就可以调用
static方法不需要创建对象 可以直接访问 这样main方法就不需要做特殊处理
static方法在静态数据区中，这样就可以直接调用不需要每次调用都要重新生成。
静态函数在调用时不用实例化
静态,一上来就装入内存

简单来说一句话总结：Main函数是入口，需要直接被加载，无法创建对象，所以static修饰。

主函数特殊之处：
1.格式固定
2.被JVM所识别和调用

public static void main(String[] args)//new String[0] JVM往main函数中传的参数
分析：
public：因为权限最大
static：不需要对象，直接用主函数所属类名调用
void：没有返回值
main：函数名，不是关键字，只是一个JVM识别的固定的名字
string[] args: 主函数参数列表，是一个数组类型的参数，而且元素都是字符串类型
接受在编译时的参数
