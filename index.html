学习笔记-2016-10-26

Posted on 2016-10-26

final
1.修饰符，可以修饰类，方法，变量
2.final修饰的类不可以被继承
3.final修饰的方法不可以被覆盖

4.final修饰的量是个常量，只能赋值一次

抽象类：
抽象函数所在的类必须也要用abstract申明

抽象只用来修饰类和方法

特点：
1.抽象类：方法只有申明没有实现时，该方法就是抽象方法，需要被abstract修饰
抽象方法必须定义在抽象类中。该类必须也被abstract修饰。
2.抽象类不可以被实例化，为什么？
因为调用抽象方法没有意义。
3.抽象类必须有其子类覆盖所有的抽象方法后，该子类才可以实例化，否则该子类依然是抽象类。

1.抽象类中有构造函数吗？
有，用于给子类对象进行初始化

2.抽象类可以不定义抽象方法吗？
可以，很少见，目的就是不让该类创建对象，AWT的适配器对象就是这种类
通常这个类中的方法有方法体，但是却没有内容。

3.抽象关键字不可以和哪些关键字共存？
private不行，因为抽象类的方法需要被覆盖
static不行，因为static不需要实例化，此时调用abstract没有意思
final不行

4.抽象类和一般类的区别
相同点：
抽象类和一般类都是用来描述事物，都在内部定了成员。
不同：
1.一般类有足够的信息描述事物
抽象类描述事物的信息可能不同
2.一般类中不能定义抽象方法，只能定非抽象方法
抽象类中可以定义抽象方法，同时也可以定义非抽象方法
3.一般类可以被实例化。
抽象类不可以被实例化。

5.抽象类是一个父类吗

是的，因为需要子类覆盖其方法后才可以对子类实例化

interface
当一个抽象类中的方法都是抽象的时候，这时可以将该抽象类用
另一种形式定义和表示，就是接口，interface
interface是特殊的抽象类

接口中常见的成员：而且这些成员都是固定的修饰符。
1.全局静态常量：public static final
2.抽象方法：public abstract

由此得出结论，接口中的成员都是公共的权限

接口不可以实例化，只能由实现了接口的子类并覆盖了接口中的所有抽象方法，该子类才可以实例化。
否则这个子类就是一个抽象类。

继承和实现区别：
类与类之间是继承关系：可选择一部分override
类与接口之间是实现关系：必须实现全部接口

多继承的关键在于方法体{}，方法体产生了调用的不确定性。

/
一个类在继承另一个类的同时，还可以实现多个接口。
接口的出现避免了单继承的局限性。
接口与接口之间是继承关系，而且接口可以多继承。 /

接口和抽象类的区别:
相同点：
都是不断向上抽取而来，都不能实例化，原因就在于没有方法体，实例化之后不知道该做什么。
不同点：
1.抽象类需要被继承，只能单继承。
接口需要被实现，而且可以多实现。
2.抽象类中可以定义抽象方法和非抽象方法，子类继承后，可以直接使用非抽象方法。
接口中只能定义抽象方法，必须有子类区实现。
3.抽象类的继承，是 is a 关系，在定义该体系的基本共性内容。

接口的实现是like 关系，在定义体系额外功能

多态

多态存在的三个必要条件
一、要有继承；
二、要有重写；
三、父类引用指向子类对象。

在代码中的体现：
父类或者接口的引用指向其子类的对象
Object ob = new Cat();//一个对象，两种形态

多态的好处：
提高了代码的扩展性，前期定义的代码可以使用后期的内容。

多态的弊端：
前期定义的内容不能使用（调用）后期子类的特有内容。

自动类型提升，无法访问特有功能，作用就是限制对特有功能的访问
专业术语：向上塑型

向下转型的目的是为了使用子类的特有方法

多态的前提：
1.必须有关系，继承，实现
2.要有覆盖

注意：
对于转型，自始至终都是子类对象在做着类型的变化。

类型判断(instanceof) //用于判断对象的具体类型，只能用于引用数据类型判断
通常在向下转型前用于代码的健壮性

多态时，成员的特点：
1.成员变量
编译时：参考引用型变量所属类中是否有调用的成员变量，有，编译通过，没有编译失败
运行时：参考引用型变量所属的类中的是否有调用成员变量，并运行该所属类中的成员变量
简单说：编译运行都参考左边

2.成员函数（非静态）
编译时：参考引用型变量所属的类中是否有调用的函数。
运行时：参考的是对象所属类中是否有调用的函数
简单说：编译看左边，运行看右边

3.静态函数
编译时：参考引用型变量所属的类中是否有调用静态方法。
运行时：参考引用型变量所属的类中是否有调用静态方法。
简单说：静态方法是不需要对象，直接用类名调用

内部类：
将一个类定义在一个类的里面

内部类访问特点：
1.内部类可以直接访问外部类中的成员
2.外部类要访问内部类，必须建立内部类的对象

一般用于类的设计。
分析事物时，发现该事物描述中还有事物，而且这个事物还在访问被描述事物的内容。
这时就是还有的事物定义成内部类来描述

如果内部类是静态的，说明就是一个外部类
如果内部类是静态的，成员是静态的。
如果内部类中定义了静态成员，该内部类也必须是静态的

内部类在局部位置上只能访问局部中被final修饰的局部变量。

匿名内部类，就是内部类的简写格式。
必须有前提：
内部类必须继承或者实现一个外部类或接口。
匿名内部类：其实就是一个匿名子类对象。
格式：
new 父类或接口(){子类内容}

new hello()
{
    void show(){
        System.out.println("hello,world");
    }
}

通常的使用场景之一：
当函数参数是接口类型时，而且接口中的方法不超过三个。

可以用匿名内部类作为实际参数进行传递。

多线程

进程：正在进行中的程序(直译)

线程：就是进程中一个负责控制程序的单元(执行路径)，称为多线程。

一个进程中至少要有一个线程。

开启多个线程是为了同时运行多部分代码。

每一个线程都有自己运行的内容，这个内容可以称为线程要执行的任务。

JVM启动时就启动了多个线程，至少有两个线程可以分析的出来。
1.执行main函数的线程
2.负责垃圾回收的线程

JVM 创建的主线程任务都定义在了主函数中。
Thread类用于描述线程，线程是需要任务的，所以Thread类也对任务的描述。
这个任务就通过Thread类中的run方法来体现，也就是说，run方法就是封装自定义线程运行任务的函数
run方法中定义的就是线程要运行的任务代码。

开启线程是为了运行指定代码，所以只有继承Thread类，并复写run
将要运行的代码写入run中。

创建线程的第一种方式，继承Thread类：(思考，万一该类已经继承了一个父类，不够灵活)
/
1.定义一个类继承Thread类
2.覆盖Thread类中的run方法
3.直接创建Thread子类对象创建线程
4.调用start方法，开启线程并调用线程的任务run方法执行。 /

静态变量什么时候用？
1.静态变量
当分析对象中所具备的成员变量的值都是相同的。
这是这个成员就可以被静态修饰。
只要数据在对象中都是不同的，就是对象的特有数据，必须存储在对象中，非静态
如果是相同的数据，对象不需要做修改，只需要使用即可，不需要存储在对象中，定义成静态。
2.静态函数（功能）
函数是否用静态修饰，就参考一点，就是该函数功能是否有访问到对象中的特有数据
简单点说，从源代码看，该功能是否需要访问非静态的成员变量，如果有需要，该功能就是非静态。
如果不需要，就可以将该功能定义成静态
当然，也可以定义成非静态，但是非静态需要被对象调用，而仅创建对象调用非静态的没有访问特有数据额方法，该对象的创建是没有意思的

静态函数只能调用静态成员变量，非静态函数则可以调用普通成员和静态成员。

构造代码块，可以给所有对象进行初始化
{} 创建对象时调用
构造函数，给对象进行针对性的初始化

静态代码块
static{}
随着类的加载而执行。而且只执行一次
作用：用于给类初始化

局部代码块（限定局部变量生成周期），构造代码块 ，静态代码块

执行顺序：静态代码块->构造代码块->构造函数

测试：
public class Person {
private String name;
private static String country;

{               //构造代码块，在创建对象前执行
    show();
}

Person(){       //创建对象时调用构造函数
    country = "cn";
}

static{         //静态代码块在类加载时执行
    country = "USA";
}

public void show(){
    System.out.println(country);
}


public static void main(String[] args){
    Person p1 = new Person();
    p1.show();
}

}

单继承：即只有一个直接飞父类
多继承：通过“多实现”的方式体现

当要使用一个继承体系时，
1.查看该体系的顶层类，了解该体系的基本功能。
2.创建体系中的最子类对象，完成功能的使用。

在子父类中，成员的特点体现（所有的成员）

当本类的成员和局部变量同名用this区分
当子父类中的成员变量同名用super区分
this和super用户很相似
this代表一个本类对象的引用，super代表一个父类的空间
子类不可访问父类的私有成员变量，要想访问，通过get、set方法

/*2.成员函数
当子父类中出现成员函数一模一样的情况，会运行子类的函数
这种现象，称为覆盖操作，这是函数在子父类中的特性。
函数两个特性，一个叫重载，一个叫覆盖。
1.重载，在同一个类中 overload，多个同名函数，通 过参数列表实现
2.覆盖，在子类中。覆盖也称为重写，override

覆盖主要实现：
1.子类方法覆盖父类方法时，子类权限必须要大于等于父类的权限
2.静态只能覆盖静态，或被静态覆盖

什么时候使用覆盖操作：
当对一个类进行子类的扩展时，子类需要保留父类的功能声明，
但是需要定义子类中该功能的特有内容时，就使用覆盖操作完成。

子父类中构造函数的特点
在子类构造对象时，发现，访问子类构造函数时，
父类也运行了。
为什么呢？
原因是：在子类的构造函数第一行有一个默认的隐式语句。super（）；
注意：
1.构造函数不能覆盖
2.构造函数

为什么子类实例化的时候要访问父类中的构造方法？
那是因为子类继承了父类，获取到了父类中的内容（属性），所以在使用父类内容之前
要先看父类是如何对自己内容进行初始化的

所以子类在构造对象时，必须访问父类中的构造函数。
为什么完成这个必须的动作，就在子类得到构造函数中加入了super（）语句

如果父类中没有定义空参数构造函数，那么子类的构造函数必须用super明确要调用父类中的哪个构造函数。
同时子类构造函数中如果使用this调用了本类构造函数是，
那么super就没有了，因为super和this都只能定义第一行，所有只能有一个。
但是可以保证的是，子类中肯定会有其他的构造函数访问父类的构造函数。

注意：
1.super 语句必须要定义在子类构造函数的第一行。因为父类的初始化动作要先完成。
2.当子父类出现重名函数时，并且该函数在父类的构造函数中调用，则在子函数的构造函数中会去优先调用子函数的该函数

基本数据类型+引用数据类型

在JVM中划分5块内存区
1.寄存器（Program Counter Register）
2.本地方法区（系统相关）
3.方法区（数据区，共享区，ShareData）
4.栈
method -> 栈
存储的都是局部变量（Var）,也在栈区
变量所属的作用域一旦结束，该变量就自动释放
5.堆
存储的是数组和对象（数组就是对象）
堆中存储实体
new 建立的对象都在堆中。
布尔值默认为假
特点：
1.每一个实体都有一个首地址值
2.堆内存中的每一个变量都有默认初始化值

定义类：定义成员变量filed（属性）和成员函数method（行为）

field：
A data member of a class. Unless specified otherwise, a field is not static.

class variable：
A data item associated with a particular class as a whole–not with particular instances of the class. Class variables are defined in class definitions.
Also called a static field. See also instance variable.

instance variable：
Any item of data that is associated with a particular object. Each instance of a class has its own copy of the instance variables defined in the class.
Also called a field. See also class variable.

local variable：
A data item known within a block, but inaccessible to code outside the block.
For example, any variable defined within a method is a local variable and can’t be used outside the method

1.
成员变量定义在类中，整个类都可以访问
局部变量定义在函数，语句，局部代码块中，只在所属区域有效

4.
成员变量都有默认初始化值

成员变量和静态变量的区别？
1.变量的生命周期不同
成员变量随着对象的创建而存在，随着对象的被回收而释放
静态变量随着类的加载而存在，随着类的消失而消失。
（一般情况下，虚拟机结束，类就消失，跟GC有关）
2.调用方式不同。
成员变量只能被对象调用。
静态变量可以被对象调用，还可以被类名调用。
3.别名不同。
成员变量也称为实例变量。
静态变量称为类变量。
4.数据存储位置不同
成员变量数据存储在堆内存中，所有也叫对象的特有数据。
静态变量数据存储在方法区的（方法区中的Runtime Constant Pool）

注意事项：
1.静态方法只能访问静态成员（非静态既可以访问静态也可以访问非静态）
2.静态方法中不能使用This或者Super
3.主函数是静态的
解释：如果main()方法没有使用static修饰符，那么编译不会出错，但是如果你试图执行该程序将会报错，提示main()方法不存在。因为包含main()的类并没有实例化（即没有这个类的对象），所以其main()方法也不会存。而使用static修饰符则表示该方法是静态的，不需要实例化即可使用。
main方法是一个程序的入口，如果写成非静态的，那么就必须实例化一个对象再来调用它，既然是入口这样肯定是不可以的；静态方法是属于类的，直接用类名就可以调用
static方法不需要创建对象 可以直接访问 这样main方法就不需要做特殊处理
static方法在静态数据区中，这样就可以直接调用不需要每次调用都要重新生成。
静态函数在调用时不用实例化
静态,一上来就装入内存

简单来说一句话总结：Main函数是入口，需要直接被加载，无法创建对象，所以static修饰。

主函数特殊之处：
1.格式固定
2.被JVM所识别和调用

public static void main(String[] args)//new String[0] JVM往main函数中传的参数
分析：
public：因为权限最大
static：不需要对象，直接用主函数所属类名调用
void：没有返回值
main：函数名，不是关键字，只是一个JVM识别的固定的名字
string[] args: 主函数参数列表，是一个数组类型的参数，而且元素都是字符串类型
接受在编译时的参数
